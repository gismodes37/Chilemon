Características

ChileMon ya no es solo una interfaz: hoy es una base funcional, instalable y preparada para crecer con orden.

Instalador guiado

Instalación pensada para usuario final, con validaciones y resumen final claro.

Configuración por nodo

Cada instalación genera su propio config/local.php sin depender del nodo del desarrollador.

Autodetección del entorno

Detecta hostname del nodo y parámetros AMI desde la instalación real de ASL/Asterisk.

Login + sesión

Acceso controlado, sesión consistente y endpoints protegidos.

SQLite estándar

Cero dependencia de MySQL/MariaDB para funcionar en el nodo.

Wrapper seguro

Uso controlado de comandos Asterisk mediante chilemon-rpt y sudoers acotado.

Conectar/Desconectar

Acciones operativas básicas con registro de eventos y base preparada para integración real.

Favoritos por usuario

Acceso rápido a nodos frecuentes con alias y descripción personalizada.

Sub-path real

Pensado para publicarse como /chilemon sin romper el entorno ASL3 existente.

Seguridad

ChileMon prioriza una base segura, predecible e instalable. La arquitectura actual se diseñó para crecer sin abrir accesos innecesarios en el nodo.

Autenticación y sesión

  • CSRF en formularios.
  • Sesión centralizada con login/logout consistentes.
  • BASE_PATH / BASE_URL dinámicos para sub-path y proxy inverso.
  • Endpoints protegidos bajo sesión/autorización.

Operación segura en el nodo

  • Wrapper seguro para comandos rpt de Asterisk.
  • sudoers acotado para www-data.
  • Configuración local no versionada: config/local.php.
  • Validación de módulos PHP/SQLite antes de inicializar ChileMon.
La base de datos, logs, respaldos y configuración local se generan en el nodo. No deben versionarse ni copiarse como parte del código fuente.

Instalación

ChileMon se instala en el nodo y publica su interfaz como /chilemon. El instalador actual detecta hostname y parámetros AMI del entorno real cuando es posible.

Requisitos

  • Debian / Raspberry Pi con ASL3
  • Apache + PHP 8.x
  • Extensiones PHP: PDO, pdo_sqlite y sqlite3
  • Git instalado en el nodo
La instalación recomendada crea configuración local por nodo y deja la DB SQLite fuera del control de versiones.

Flujo recomendado

$ sudo apt update
$ sudo apt install -y git
$ cd /opt
$ sudo git clone https://github.com/gismodes37/Chilemon.git chilemon
$ cd /opt/chilemon
$ sudo bash install/install_chilemon.sh
El instalador guía la configuración básica y deja el acceso final listo como https://NOMBRE_DEL_NODO/chilemon cuando el entorno ya publica HTTPS.
Tip: mantén una única ruta oficial en Apache hacia /opt/chilemon/public para evitar conflictos y confusión en el nodo.

Arquitectura

Componentes actuales

  • UI: public/views/*
  • Auth: login/logout + sesión + CSRF
  • API: public/api/*
  • DB: SQLite en data/chilemon.sqlite
  • Schema: install/sql/schema.sql
  • Config: config/app.php + config/local.php
  • Instalador: install/install_chilemon.sh
  • Inicialización DB/usuarios: bin/install.php
  • Wrapper Asterisk: /usr/local/bin/chilemon-rpt

FAQ

ChileMon no fue creado para reemplazar ni desplazar herramientas existentes dentro del ecosistema ASL3. Reconoce el valor histórico y operativo de proyectos como Supermon y otros sistemas que han servido a la comunidad durante años.

ChileMon nace como un desarrollo independiente, construido desde cero, con una arquitectura propia, enfoque en seguridad, instalación ordenada y evolución controlada.

La idea es aportar una alternativa moderna, sostenible y mantenible, diseñada para evolucionar junto a ASL3 sin depender de estructuras heredadas.

Porque simplifica la instalación, reduce dependencias externas y funciona muy bien en Raspberry Pi y nodos ASL3 donde la prioridad es estabilidad y bajo mantenimiento.
