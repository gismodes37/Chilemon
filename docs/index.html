ChileMon

Dashboard estilo Supermon para nodos ASL3, con una base moderna: autenticación, favoritos, registro de eventos y conexión/desconexión. Diseñado para instalarse como sub-path: /chilemon.

Seguridad (cambios recientes)

ChileMon prioriza una base segura y predecible. Estos cambios consolidan el “core” antes de crecer a funciones de voz/control.

Autenticación y sesión

  • CSRF en formularios (token de sesión).
  • Sesión centralizada (inicio/cierre consistente, logout con redirect correcto a /chilemon/login.php).
  • Rutas dinámicas: BASE_PATH y BASE_URL calculados (compatible con sub-path y reverse proxy).
  • Hardening de sesión: preparado para fijar flags (HttpOnly/SameSite/secure) sin romper ASL.

Base de datos y API

  • PDO SQLite como única DB soportada por defecto.
  • Singleton de conexión (una sola fuente de verdad).
  • Schema upgrade-safe (aplicación repetible del schema para upgrades).
  • Endpoints protegidos (API bajo sesión/autorización).
  • DB fuera del repo: se crea por instalador/CLI; no se versiona.
Nota para dev local: si aparecen warnings de ini_set() en sesión, es señal de que las directivas de sesión deben definirse antes de iniciar la sesión. Esto se resolverá como parte de “paridad local vs ASL3”.

Instalación

Instalación típica en Raspberry Pi con ASL3. ChileMon vive en /opt/chilemon y se expone como /chilemon.

Requisitos

  • Apache + PHP 8.x
  • Extensiones PHP: pdo_sqlite y sqlite3
  • Permisos de escritura en /opt/chilemon/data y /opt/chilemon/logs
La DB SQLite no se versiona. Se crea y actualiza vía scripts/instalador.

Pasos (resumen)

$ cd /opt
$ sudo git clone https://github.com/gismodes37/Chilemon.git chilemon
$ cd chilemon
$ # ejecutar instalador/CLI según README
Apache debe mapear /chilemon → /opt/chilemon/public.
Tip: evita duplicar Alias/VHost en Apache. Mantén una ruta oficial hacia public/.

Arquitectura (hoy)

Componentes

  • UI: public/views/* (partials + dashboard)
  • Auth: login/logout + sesión + CSRF
  • API: public/api/* (protegida)
  • DB: SQLite en data/chilemon.sqlite (fuera del repo en producción)
  • Schema: install/sql/schema.sql (upgrade-safe)
  • Config: config/app.php (BASE_PATH/BASE_URL dinámicos)

FAQ

ChileMon no fue creado para reemplazar ni desplazar herramientas existentes dentro del ecosistema ASL3. Reconoce el valor histórico y operativo de proyectos como Supermon y otros sistemas que han servido a la comunidad durante años.

La evolución y modernización de ASL3 representa un hito importante dentro de AllStarLink. No solo mejora aspectos técnicos y de seguridad, sino que consolida una base tecnológica que proyecta estabilidad y permanencia en el tiempo. Esa renovación estructural demuestra que el ecosistema continúa avanzando, y al mismo tiempo abre espacio para nuevas propuestas que acompañen ese crecimiento.

ChileMon nace en ese contexto de madurez tecnológica. No es una adaptación ni una modificación de proyectos existentes. Es un desarrollo completamente independiente, construido desde cero, con su propia arquitectura, estructura de seguridad y modelo de funcionamiento. No reutiliza código externo ni deriva de implementaciones previas; su diseño responde a una visión técnica orientada a mantenibilidad, claridad estructural y evolución controlada.

La inspiración proviene de conceptos y buenas prácticas observadas en el ecosistema, pero la implementación es original. Esto permite que ChileMon pueda crecer de manera ordenada, incorporar nuevas funcionalidades progresivamente y mantener estabilidad sin depender de decisiones externas.

El propósito es aportar una alternativa sólida, moderna y sostenible, diseñada para evolucionar junto a ASL3 y contribuir de manera responsable al desarrollo continuo de la comunidad global de radioaficionados.

Portabilidad y simplicidad: cero servicios extra. Ideal para Raspberry y entornos ASL3.
