Características

ChileMon ya no es solo una idea o una base técnica: el release actual entrega una primera versión funcional y usable en nodos reales.

Instalador automático

Instalación guiada con validaciones, creación de estructura, wrapper, configuración local y resumen final de acceso.

Configuración por nodo

Cada instalación genera su propio config/local.php según el entorno real del nodo.

Autodetección del entorno

Detecta hostname del nodo y parámetros AMI desde la instalación real de ASL/Asterisk cuando están disponibles.

Login + sesión

Acceso controlado, endpoints protegidos y una base segura para administración del dashboard.

SQLite estándar

Sin MySQL ni MariaDB. La instalación queda simple, portable y fácil de mantener en el nodo.

Wrapper seguro

Uso controlado de comandos Asterisk mediante chilemon-rpt y sudoers limitado a lo necesario.

ASL & EchoLink

Conexión y desconexión funcional desde el dashboard sobre nodos AllStarLink y EchoLink (vía prefijo 8 o 3).

Favoritos por usuario

Acceso rápido a nodos frecuentes con alias y descripción personalizada por usuario autenticado.

Actividad reciente

Registro de eventos básicos para acompañar la operación del nodo y facilitar el monitoreo.

Estado actual: release v0.3.0 funcional. ChileMon ya integra favoritos con alias personalizados, monitoreo RX/TX y se instala en nodos reales de forma automática.

Seguridad

ChileMon está construido para crecer con una base segura, controlada y predecible dentro de un nodo ASL3.

Autenticación y sesión

  • CSRF en formularios.
  • Sesión centralizada con login/logout consistentes.
  • BASE_PATH / BASE_URL dinámicos para sub-path y despliegues reales en el nodo.
  • Endpoints protegidos bajo sesión/autorización.

Operación segura en el nodo

  • Wrapper seguro para comandos rpt de Asterisk.
  • sudoers acotado para www-data.
  • Configuración local no versionada en config/local.php.
  • Validación de SQLite / PDO antes de inicializar la aplicación.
La base de datos, logs, respaldos y configuración local se generan en el nodo. No deben subirse al repositorio ni tratarse como parte del código fuente.

Instalación

ChileMon se instala en el nodo y publica su interfaz como /chilemon. El instalador actual detecta hostname y parámetros AMI del entorno real cuando es posible.

Requisitos y entornos probados

  • Sistemas probados: Raspberry Pi 3B+ con ASL3 y Máquina Virtual Proxmox con Debian 13 + ASL3
  • Apache + PHP 8.x
  • Extensiones PHP: PDO, pdo_sqlite y sqlite3
  • Git instalado en el nodo
La instalación recomendada crea configuración local por nodo, inicializa SQLite y deja el acceso web operativo bajo /chilemon.

Flujo recomendado

# Clonar versión estable e instalar
$ sudo git clone -b v0.1.0 https://github.com/gismodes37/Chilemon.git /opt/chilemon
$ cd /opt/chilemon && sudo bash install/install_chilemon.sh
El instalador deja un resumen final con dirección de acceso, nodo local, usuario AMI detectado y datos relevantes de la instalación.
Mantén una única ruta oficial en Apache hacia /opt/chilemon/public para evitar conflictos en el nodo.

Arquitectura

Componentes actuales

  • UI: public/views/*
  • Auth: login/logout + sesión + CSRF
  • API: public/api/*
  • DB: SQLite en data/chilemon.sqlite
  • Schema: install/sql/schema.sql
  • Config: config/app.php + config/local.php
  • Instalador: install/install_chilemon.sh
  • Inicialización: bin/install.php
  • Wrapper Asterisk: /usr/local/bin/chilemon-rpt
